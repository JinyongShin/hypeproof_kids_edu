# Test Plan

## Instructions

This file tracks your TDD test plan following Kent Beck's principles.

- `[ ]` = Test not started
- `[x]` = Test completed
- Use `/tdd go` to automatically execute the next unmarked test (Red → Green → Refactor cycle)
- Use `/tdd next` to view the next test without executing
- Use `/tdd mark` to mark the current test as done
- Use `/tdd plan-add` to add new tests to this plan

## TDD Workflow

1. Write a failing test (Red)
2. Implement minimum code to pass (Green)
3. Refactor while keeping tests green
4. Commit and repeat
