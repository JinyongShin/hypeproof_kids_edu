# ✨ 나만의 마법 게임 만들기: 지휘관의 탄생
**2026.5.5 (화) 13:30–15:30 · 국립암센터 소아암 병동**

---

## 스토리 아크

```
병원 = 지시받는 곳
        ↓
오늘은 내가 지시하는 자
        ↓
내 캐릭터 창조 → 내 세계 창조 → 세상에 공개
        ↓
증거(QR + 사원증)를 손에 쥐고 복귀
```

---

## 사전 정의

> **"게임 빌드"의 실제 의미**
> 아이가 묘사한 캐릭터 + 세계 + 규칙을 AI가 받아 **게임 타이틀 카드 페이지**로 즉시 생성.
> 실제 조작 게임이 아님 — 캐릭터 이미지, 세계관 텍스트, 승리 조건, 개발자 이름이
> 한 화면에 완성되는 것. 이것이 "플레이 가능한 게임"처럼 느껴지도록 UI 설계 필요.

---

## ⚠️ 진행 전 파악된 문제점 5가지

### 🔴 1. "즉시 플레이 가능한 게임 빌드" — 기술이 존재하지 않음

원본 커리큘럼에 "아이가 시나리오를 정의하면 실제 플레이 가능한 게임이 즉시 생성된다"고 되어 있으나,
이런 기술은 현재 존재하지 않음.

- AI로 생성 가능한 것: 텍스트, 이미지, 코드 — "플레이 가능한 게임"이 아님
- 커리큘럼 전체 설득력이 이 한 줄에 달려 있으므로 구현 방식 사전 확정 필수
- **해결**: 게임 타이틀 카드 페이지(이미지 + 텍스트 레이아웃)로 재정의

### 🔴 2. 40명 동시 진행 — 병동 집합 허가 문제

- 소아암 병동은 면역저하 환자가 많아 집단 집합 자체를 병원이 허가하지 않을 수 있음
- 40명 중 그날 컨디션 불량, 수액 중, 격리 중인 아이 포함 가능
- **병원 사전 승인 없이 진행 계획 수립 불가**

### 🔴 3. 동아일보 취재 — 미디어 촬영 동의 문제

- 소아암 환자: 미성년자 + 의료 환경 → 촬영 동의 기준 매우 엄격
- 보호자 개별 서면 동의 없이 촬영 불가
- 병원 홍보팀 통해 사전 협의 필수
- 당일 거부 아이 발생 시 행사 흐름 차질 가능

### 🟡 4. 연령 범위 미정 — 커리큘럼이 맞춰진 나이 불명확

- 소아암 = 0~18세
- "문제를 정의하라", "지휘관의 마음가짐" → 초등 저학년·미취학 아동에게 추상적
- 타이핑 자체가 불가한 나이도 포함될 수 있음
- **대상 연령대 먼저 확정 후 커리큘럼 세부 조정 필요**

### 🟡 5. 기기 40대 + 안정적 Wi-Fi — 준비 주체 미확정

- 40대 동시 접속 시 병원 Wi-Fi 부하 불확실
- AI API 동시 요청 40건 → 응답 지연 또는 오류 가능
- 기기 준비 주체(병원 / 행사팀 / 개인) 명시 필요

## 🕒 블록 1: 지휘관의 선언 (13:30–13:45, 15분)

**포맷**: 진행자 주도, 대형 스크린, 전체 관람

### 진행자 오프닝 스크립트

> "오늘 이 시간엔 선생님도, 부모님도 명령 못 해요.
> 여기서 명령하는 사람은 여러분뿐이에요.
> 그런데 AI는 멍청해서 — 여러분이 정확하게 말해줘야만 움직여요."

### 시연 순서 (스크린에서 직접)

| 순서 | 입력 내용 | 결과 | 예상 반응 |
|------|-----------|------|-----------|
| ① 나쁜 명령 | "멋진 기사 만들어줘" | 평범하고 뻔한 결과 출력 | "에이~" |
| ② 좋은 명령 | "은빛 갑옷을 입고 왼쪽 눈에 흉터가 있는 여자 기사, 밤하늘 별빛 아래 서 있어" | 선명하고 구체적인 결과 | "와!" |
| ③ 선언 | — | "더 구체적으로 말할수록 AI가 굴복해요. 여러분이 지휘관입니다." | — |

**시간 배분**: 오프닝 3분 + 시연 8분 + 전환 안내 4분

> ⚠️ **리스크 대응**: 인터넷 지연 시 → 사전 캡처된 결과 이미지 슬라이드로 즉시 대체

## 🕒 블록 2: 아바타 소환 (13:45–14:05, 20분)

**포맷**: 개인 기기, 개별 진행

### 웹앱 화면 흐름

```
[화면 1]
"당신의 캐릭터는 어떻게 생겼나요?"
힌트: 어떤 옷? 눈 색깔? 들고 있는 것?
→ 자유 입력창

[화면 2]
AI 유도 질문 (미리 설계된 문장, 실시간 AI 아님)
예: "이 캐릭터의 가장 강한 능력은 뭔가요?"
→ 입력

[화면 3]
"당신의 캐릭터가 소환되었습니다"
→ 캐릭터 이미지 + 묘사 텍스트 카드 생성
→ [다음 단계로] 버튼
```

### 진행자 역할

| 상황 | 진행자 행동 |
|------|------------|
| 아이가 막힌 경우 | "지휘관님, 이 캐릭터 머리카락은 어떤 색이에요?" |
| "그냥 멋있게" | "멋있다는 게 칼을 드는 건가요, 마법을 쓰는 건가요?" |
| 타이핑 느린 아이 | 옆에서 불러주면 아이가 직접 입력 (대신 치지 않음) |
| 이미 완료한 아이 | 옆 친구 작품 구경 유도 |

> ❌ 절대 금지: 대신 입력해주기, 정답 알려주기
> ✅ 허용: 질문으로 유도, 선택지 제시

**타이밍**: 대부분 10~15분 완료. 남은 시간은 완성된 아이들끼리 자유롭게 구경.

## 🕒 블록 3: 세계 구축 (14:05–14:25, 20분)

**포맷**: 개인 기기, 개별 진행

### 웹앱 화면 흐름

```
[화면 1]
"당신의 캐릭터가 사는 세계는 어디인가요?"
힌트 버튼: [우주] [바닷속] [마법 숲] [미래 도시] [직접 입력]
→ 선택 또는 자유 입력

[화면 2]
"이 세계에서 이기려면 무엇을 해야 하나요?"
예시: "바이러스를 피해 별을 10개 모은다"
→ 입력

[화면 3]
"게임이 만들어지고 있습니다..." (로딩 애니메이션 3초)

[화면 4 — 게임 타이틀 카드 완성]
┌──────────────────────────────┐
│  [게임 제목]                  │
│  [세계 배경 이미지]            │
│  캐릭터: [이름 / 묘사]         │
│  미션: [승리 조건]             │
│  총괄 개발자: OOO             │
└──────────────────────────────┘
→ [완성] 버튼 → 서버 저장 + 갤러리 자동 반영
```

### 설계 포인트

- **힌트 버튼 제공 이유**: 아이디어 없어 멈추는 것 방지. 선택지를 눌러도 "내가 선택한 것"이므로 창조감 유지.
- **"총괄 개발자" 이름**: 앞서 로그인 시 입력한 이름 자동 반영.
- **완성 카드**: 서버에 자동 저장 → 갤러리 슬라이드쇼에 즉시 반영.
