## 1. 회의 목적

이번 회의는 SK바이오팜 임직원 가족을 대상으로 한 AI 교육 프로그램의 운영 가능성을 검토하고, HypeProof Lab이 제공할 수 있는 교육 방향, 운영 방식, 강사 구조, 시스템 요구사항, 비용 구조, 향후 확장 가능성을 논의하기 위해 진행되었습니다.

이번 프로그램은 단순한 AI 체험 수업이나 코딩 교육이 아니라, 아이들이 AI와 대화하며 자신만의 게임을 완성해가는 **AI 창작 프로젝트 수업**으로 정의합니다.

HypeProof Lab 입장에서는 이번 프로그램을 향후 기업 임직원 가족 교육, 대치동형 프리미엄 교육, 해외 캠프형 교육, 강사 양성 프로그램, AI Competition으로 확장하기 위한 첫 번째 교육 프로덕트 파일럿으로 볼 수 있습니다.

| 항목       | 내용                     |
| -------- | ---------------------- |
| 장소       | SK바이오팜 10층 내부 카페       |

## 3. 교육 일정 및 운영 방식

교육은 6~7월 중 토요일을 활용하여 Biweekly 형태로 운영하는 방향이 논의되었습니다.

중요한 점은 **8시간이 한 수강생이 듣는 연속 과정이 아니라는 점**입니다.
기본 수업 단위는 4시간이며, 하루에 2개 그룹을 운영할 수 있어 하루 총 운영 시간이 8시간이 되는 구조입니다.

| 항목          | 내용                      |
| ----------- | ----------------------- |
| 기본 수업 단위    | 4시간                     |
| 하루 운영 가능 그룹 | 2개 그룹                   |
| 하루 총 운영 시간  | 4시간 + 4시간 = 총 8시간       |
| 그룹 분리 기준    | 학년 / 연령대 / 수준           |
| 예시          | 초등 3학년 4시간 + 초등 4학년 4시간 |
| 운영 기간       | 6~7월 토요일 Biweekly 검토    |

예를 들어 오전에는 초등 3학년 그룹을 4시간 운영하고, 오후에는 초등 4학년 그룹을 4시간 운영할 수 있습니다.
또는 주차별로 다른 학년 또는 수준별 그룹을 배치하여 여러 연령대가 순차적으로 참여할 수 있도록 구성할 수 있습니다.

## 5. 프로그램 기본 방향

이번 프로그램의 핵심은 **게임 만들기**입니다.

다만 이 수업은 코딩을 배우거나, 기술 구현을 직접 다루는 수업이 아닙니다.
아이들은 학년별로 사전에 설계된 제작 환경과 방향성 안에서 AI와 대화하며 자신만의 게임을 완성합니다.

아이들은 자신이 만들고 싶은 게임의 세계관, 캐릭터, 규칙, 미션, 난이도, 스토리를 구상하고, AI와의 대화를 통해 이를 결과물로 발전시킵니다.

즉 수강생은 개발자가 아니라 **게임 기획자, 스토리 디렉터, AI 협업자, 테스트 플레이어**의 역할을 경험합니다.

핵심 정의는 다음과 같습니다.

> HypeProof Lab의 수업은 아이들이 AI와 함께 자신만의 게임을 만들어가는 창작형 프로젝트 수업입니다.
> 코딩을 직접 가르치는 것이 아니라, AI와 대화하며 자신의 아이디어를 구조화하고, 스토리를 녹이고, 결과물을 개선해가는 경험을 제공합니다.

---

## 6. HypeProof Lab 강의의 차별점

HypeProof Lab의 차별점은 “AI 사용법을 알려준다”가 아닙니다.
이미 많은 초등 AI 방과후 수업과 기업 사회공헌형 AI 교육 프로그램이 AI 체험, 코딩, 로봇, 생성형 AI 활용을 다루고 있습니다.

HypeProof Lab은 이와 다르게, 아이들이 AI와 협업하며 하나의 창작물을 완성하는 경험에 집중합니다.

| 구분     | 일반 AI 방과후 수업         | HypeProof Lab                    |
| ------ | -------------------- | -------------------------------- |
| 교육 목적  | AI 체험, 코딩 흥미, 디지털 소양 | AI와 함께 자기 아이디어를 게임으로 완성          |
| 수강생 역할 | 학습자, 체험자             | 기획자, 디렉터, 창작자, 테스트 플레이어          |
| 핵심 활동  | 도구 사용, 코딩 실습, 교구 체험  | 스토리 구상, 세계관 설계, AI와 대화, 결과물 개선   |
| 학습 방식  | 개념 설명 → 실습           | 상상 → 구조화 → AI와 협업 → 검증 → 개선 → 발표 |
| 결과물    | 수업 중 활동 결과           | 아이의 아이디어와 스토리가 반영된 게임            |
| 후속 확장  | 재수강 중심               | 수료증, 성장 리포트, Competition, 강사 양성  |

정리하면 HypeProof Lab의 차별점은 다음 세 가지입니다.

첫째, **AI와 협업하는 창작 경험**입니다.
아이들은 AI에게 단순히 답을 받는 것이 아니라, 자신의 생각을 설명하고, 결과를 확인하고, 다시 개선 방향을 제시합니다.

둘째, **게임이라는 몰입도 높은 결과물**입니다.
게임은 아이들이 가장 쉽게 몰입할 수 있는 매체이며, 그 안에 스토리, 규칙, 목표, 실패, 보상, 난이도, 사용자 경험이 모두 포함됩니다.

셋째, **HypeProof Essence를 자연스럽게 배우는 구조**입니다.
아이들은 게임을 만드는 과정에서 질문하기, 상상하기, 구조화하기, 표현하기, 검증하기, 피드백하기, 반복 개선하기, 발표하기를 경험합니다.

## 7. 커리큘럼 설계 방향

이번 SK바이오팜 임직원 가족 대상 AI 교육 프로그램은 **게임 만들기를 중심으로 한 AI 창작 프로젝트 수업**으로 설계합니다.

수업은 아이들이 코드를 배우거나 기술 구조를 이해하는 방향이 아닙니다. 학년별로 사전에 설계된 제작 환경과 방향성이 준비되어 있으며, 아이들은 그 안에서 AI와 대화하며 자신만의 게임을 완성해갑니다.

아이들은 다음과 같은 질문을 스스로 던지고 구체화합니다.

| 질문                  | 의미     |
| ------------------- | ------ |
| 어떤 세계에서 벌어지는 게임인가?  | 세계관    |
| 주인공은 누구인가?          | 캐릭터    |
| 게임의 목표는 무엇인가?       | 미션     |
| 어떤 규칙이 있어야 재미있는가?   | 게임 룰   |
| 어떤 장애물이나 도전이 필요한가?  | 난이도    |
| 친구들이 플레이했을 때 재미있을까? | 사용자 경험 |
| 더 좋아지려면 무엇을 바꿔야 할까? | 개선 방향  |

이 과정을 통해 수강생은 AI 시대에 필요한 핵심 역량을 자연스럽게 익히게 됩니다.

## 8. 4시간 클래스 구성안

기본 수업 단위는 4시간입니다.

| 시간        | 내용                     | 산출물       |
| --------- | ---------------------- | --------- |
| 0:00~0:30 | 오늘 만들 게임의 방향 이해하기      | 수업 목표 이해  |
| 0:30~1:00 | 나만의 게임 아이디어 정하기        | 게임 컨셉     |
| 1:00~1:40 | 캐릭터, 세계관, 스토리 구체화      | 게임 스토리 초안 |
| 1:40~2:20 | AI와 대화하며 게임 완성해가기      | 1차 결과물    |
| 2:20~3:00 | 직접 플레이하며 재미 요소와 문제점 찾기 | 개선 아이디어   |
| 3:00~3:30 | AI에게 개선 방향 설명하기        | 개선된 게임    |
| 3:30~4:00 | 발표 및 친구 게임 플레이         | 최종 발표     |

이 구조에서 중요한 것은 아이들이 기술을 직접 다루는 것이 아니라, **자기 생각을 AI에게 잘 설명하고, 결과물을 관찰하고, 더 나은 방향으로 개선하는 과정**입니다.

## 9. 학년별 운영 방향

모든 클래스는 게임 만들기를 중심으로 운영합니다.
다만 학년별로 게임의 복잡도, 스토리 구성 수준, 미션 설계 방식, 발표 방식은 다르게 설계합니다.

### 9.1 초등 저학년

| 항목     | 내용                      |
| ------ | ----------------------- |
| 대상     | 초등 3~4학년                |
| 핵심 목표  | 캐릭터와 스토리가 있는 간단한 게임 완성  |
| 아이의 역할 | 캐릭터 상상, 이야기 만들기, 규칙 정하기 |
| 집중 요소  | 상상력, 표현력, 자기 아이디어 말하기   |
| 최종 결과  | 자신만의 캐릭터와 이야기가 담긴 게임    |

초등 저학년은 복잡한 규칙보다는 캐릭터, 배경, 이야기, 간단한 목표 설정에 집중합니다.

예시 방향성은 다음과 같습니다.

* 내가 만든 캐릭터가 모험을 떠나는 게임
* 주인공이 보물을 찾는 게임
* 동물이 친구를 구하는 게임
* 우주에서 별을 모으는 게임

---

### 9.2 초등 고학년

| 항목     | 내용                        |
| ------ | ------------------------- |
| 대상     | 초등 5~6학년                  |
| 핵심 목표  | 규칙, 점수, 미션, 난이도가 있는 게임 완성 |
| 아이의 역할 | 룰 설계, 난이도 조절, 재미 요소 구상    |
| 집중 요소  | 구조화, 판단력, 개선력             |
| 최종 결과  | 규칙과 목표가 분명한 게임            |

초등 고학년은 단순한 스토리에서 한 단계 더 나아가, 플레이어가 무엇을 해야 하는지, 어떻게 이기거나 실패하는지, 어떤 점이 재미있는지를 설계하게 합니다.

예시 방향성은 다음과 같습니다.

* 제한 시간 안에 목표를 달성하는 게임
* 장애물을 피하며 점수를 얻는 게임
* 선택에 따라 결과가 달라지는 게임
* 퀴즈나 미션을 해결하는 게임

---

### 9.3 중등 이상

| 항목     | 내용                             |
| ------ | ------------------------------ |
| 대상     | 중등 이상                          |
| 핵심 목표  | 스토리, 규칙, 미션, 사용자 경험이 결합된 게임 완성 |
| 아이의 역할 | 게임 디렉터, 기획자, 테스트 플레이어          |
| 집중 요소  | 문제정의, 기획력, 검증력, 발표력            |
| 최종 결과  | 기획 의도와 개선 과정이 설명 가능한 게임        |

중등 이상은 게임의 완성도뿐 아니라, 왜 이런 게임을 만들었는지, 어떤 사용자를 생각했는지, 무엇을 개선했는지까지 설명하는 데 집중합니다.

예시 방향성은 다음과 같습니다.

* 선택형 스토리 게임
* 퀴즈 기반 미션 게임
* 난이도 변화가 있는 도전형 게임
* 사회적 메시지나 관심 주제가 담긴 게임

## 10. HypeProof Essence와의 연결

이번 수업의 핵심은 게임을 만드는 과정에서 HypeProof Lab의 Essence를 자연스럽게 경험하게 하는 것입니다.

| 과정    | 아이들이 하는 일                 | 배우는 Essence  |
| ----- | ------------------------- | ------------ |
| 상상하기  | 어떤 게임을 만들지 떠올림            | 상상력, 질문하기    |
| 구조화하기 | 캐릭터, 세계관, 규칙, 목표를 정리함     | 기획력, 정리하기    |
| 표현하기  | AI에게 원하는 것을 설명함           | 표현력, 프롬프트 감각 |
| 확인하기  | 결과물을 직접 플레이함              | 관찰력, 검증력     |
| 개선하기  | 부족한 부분을 찾아 수정 방향을 정함      | 비판적 사고, 피드백  |
| 반복하기  | AI와 다시 대화하며 더 나은 결과로 발전시킴 | 반복 개선        |
| 발표하기  | 내가 만든 게임의 의도와 특징을 설명함     | 발표력, 공유 능력   |

즉 게임은 단순한 흥미 요소가 아니라, AI 시대에 필요한 사고방식과 표현 방식을 훈련하기 위한 매개체입니다.

## 11. 기술 및 운영 구조

외부 제안서에서는 기술 구현 세부사항을 전면에 내세우지 않습니다.
다만 내부 운영 관점에서는 안정적인 수업 진행을 위해 사전 제작 환경이 필요합니다.

본 프로그램은 사전에 준비된 설치 및 실행 환경을 기반으로 운영합니다. 해당 환경 안에는 학년별 수업 방향성, 제작 흐름, AI와의 대화 구조, 결과물 완성을 위한 기본 조건이 포함됩니다.

수강생은 복잡한 설치나 기술 구현에 집중하지 않고, 준비된 환경에서 AI와 대화하며 자신의 게임을 완성합니다.

| 구분      | 내용                        |
| ------- | ------------------------- |
| 사전 준비   | 수업용 환경 사전 구성              |
| 학생 경험   | AI와 대화하며 아이디어를 게임으로 구체화   |
| 강사 역할   | 좋은 질문, 명확한 설명, 개선 방향을 코칭  |
| 보조강사 역할 | 환경 확인, 진행 보조, 개별 피드백      |
| 목표      | 기술 복잡도는 낮추고 창작 경험은 높이는 구조 |

중요한 점은 수강생에게 기술적 복잡도를 노출하지 않으면서도, 실제로는 안정적으로 결과물이 완성될 수 있도록 운영 환경을 표준화해야 한다는 점입니다.

## 14. 결과물 및 공유 방식

수업의 최종 결과물은 아이가 직접 만든 게임입니다.
다만 외부 커뮤니케이션에서는 기술적 공유 방식보다, “아이의 아이디어와 스토리가 반영된 결과물”이라는 점을 강조합니다.

| 항목     | 내용                      |
| ------ | ----------------------- |
| 최종 결과물 | 아이가 AI와 함께 완성한 게임       |
| 포함 요소  | 캐릭터, 세계관, 규칙, 미션, 스토리   |
| 발표 방식  | 자신의 게임 소개 및 친구들과 플레이    |
| 공유 방식  | 내부적으로 별도 검토             |
| 외부 표현  | 공유 가능한 결과물, 수업 결과물, 창작물 |

기술적인 공유 방식은 내부적으로 검토하되, 제안서에서는 너무 기술 중심으로 보이지 않도록 표현합니다.

정리 문구는 다음과 같습니다.

이 문구는 향후 결과물 홍보, 후기 콘텐츠, 포트폴리오 활용 시 필요합니다.

## 16. 강사 투입 구조

회의에서는 총 7명이 참여하는 운영 구조가 논의되었습니다.

| 역할      | 인원 | 역할                                  |
| ------- | -: | ----------------------------------- |
| 메인강사    | 1명 | 전체 강의 진행, 수업 흐름 관리, 핵심 메시지 전달       |
| 보조강사    | 6명 | 개별 진행 보조, 질문 대응, 환경 지원, 아이디어 구체화 지원 |
| 총 투입 인원 | 7명 | 최대 20가족 커버                          |

보조강사 1명당 최대 5명까지 커버 가능하다고 논의했습니다.
실제 20가족 기준 보조강사 6명이 투입되면 보조강사 1명당 약 3~4가족을 담당하게 되어 안정적인 운영이 가능합니다.

아이들 대상 AI 창작 수업에서는 아이디어가 막히거나, AI에게 어떻게 말해야 할지 모르는 순간이 자주 발생할 수 있습니다. 따라서 보조강사의 역할은 단순 기술 지원이 아니라, 아이가 자신의 생각을 말로 풀어내도록 돕는 코치 역할에 가깝습니다.

## 22. SK바이오팜 제안서 핵심 메시지

> HypeProof Lab의 프로그램은 아이들이 AI와 함께 자신만의 게임을 만들어가는 창작형 프로젝트 수업입니다.
>

>
> 수업의 목적은 코딩 기술을 가르치는 것이 아니라, AI 시대에 필요한 상상력, 기획력, 표현력, 검증력, 개선력을 게임 제작 과정을 통해 자연스럽게 익히는 것입니다.

> 기존 AI 교육이 아이들에게 AI를 소개하는 수업이라면, HypeProof Lab은 아이들이 AI와 함께 자신의 세계를 게임으로 완성해보는 창작형 프로젝트 수업입니다.

## 23. 최종 정리

이번 SK바이오팜 프로그램은 HypeProof Lab 입장에서 중요한 파일럿입니다.

핵심은 “AI를 가르친다”가 아니라, **AI와 함께 창작하는 경험을 설계한다**는 것입니다.

아이들은 코딩 문법이나 기술 구조를 배우는 것이 아니라, 자신이 만들고 싶은 세계를 상상하고, AI에게 설명하고, 결과물을 확인하고, 다시 개선하는 과정을 경험합니다.

게임은 단순한 결과물이 아니라, 아이들이 AI 시대의 핵심 역량을 배우기 위한 매개체입니다.

따라서 이번 제안에서 강조해야 할 것은 다음 세 가지입니다.

첫째, **게임 중심의 높은 몰입도**입니다.
아이들이 가장 쉽게 빠져들 수 있는 게임을 통해 AI와의 협업을 경험합니다.

둘째, **AI와 대화하며 생각을 구조화하는 경험**입니다.
아이들은 자신의 아이디어를 말로 풀어내고, AI에게 전달하고, 결과를 확인하며 개선합니다.

셋째, **HypeProof Essence 기반의 확장 가능성**입니다.
이번 수업은 단발성 강의가 아니라, 수료증, 성장 리포트, AI Competition, 강사 양성, 캠프형 교육으로 확장될 수 있는 교육 프로덕트의 시작점입니다.
