# 커리큘럼 v0.3 — 나만의 마법 세계를 만들자

2026-05-05 국립암센터 소아암 병동 어린이 대상 AI 창작 수업 커리큘럼. 작성자: [[bongho-tae]] (봉호), [[jiwoong-kim]] (지웅). 2026-04-19 전달.

> [!note] 버전 맥락
> 이 커리큘럼은 [[pilot-5-5-milestones]] 기준 2026-04-19 마감 산출물. [[bongho-tae]]가 4/19 목표로 작성한 v0.3이며, 2026-04-21 Jay·JY 미팅에서 리뷰 예정 ([[stack-decision-after-curriculum]] 게이트 연동).

> "내가 존재하는 세계가 있다"

소아암 병동의 아이가 처음으로 통제하는 세계. 내가 고르고, 내가 만들고, 내 이름이 새겨지고, 내가 인정받는다.

## 6블록 타임테이블

### 블록 1 — 오프닝 (13:30–13:45, 15분)

- **형식**: 전체 · 대형 스크린
- **활동**:
  - 데모: 아이 이름으로 즉석 "OO의 마법 정원" 생성
  - 조별 자기소개 (좋아하는 동물/색깔)

> [!note] 와우 포인트
> 병원에서 이름 불릴 때 = "주사 맞자"뿐이던 아이가, 마법의 주인공으로 이름이 불린다. 엄마가 옆에서 보는데, 내 아이가 환자가 아니라 작가다.

**AI 호출**: 없음 (퍼실리테이터 데모)

---

### 블록 2 — 캐릭터 만들기 (13:45–14:05, 20분)

- **형식**: 조별 개인 기기 · 저학년 버튼 / 고학년 자유입력
- **활동**:
  - 버튼 선택: [꽃의 마법사 / 별의 마법사 / 바람의 마법사] + 고학년 "직접 마법 주문 쓰기"
  - 이름 · 색깔 선택 → AI 즉시 캐릭터 카드 생성
  - 빠른 아이: 카드 터치 시 별가루 파티클 인터랙션

> [!note] 와우 포인트
> 아이가 3번 버튼 누르면 5초 뒤 자기 캐릭터가 화면에 나타난다. 병원에서는 아무것도 자기가 정하는 게 없는데, 지금은 내가 정하면 세상이 바뀐다.

**AI 호출 1**: 캐릭터 카드 생성 (이름 + 마법사 유형 + 색깔 → 캐릭터 카드 이미지+텍스트)

**완성 보장 (R4)**: 15분 경과 미완성 → 기본 선택지 자동 완성

---

### 블록 3 — 마법 세계 만들기 (14:05–14:25, 20분)

- **형식**: 개인 · 저학년 버튼 / 고학년 자유입력
- **활동**:
  - 세계 선택: [반짝이는 숲 / 무지개 바다 / 구름 위 마을]
  - 활동 선택: [꽃밭 가꾸기 / 별 모으기 / 친구 찾기]
  - AI가 아이 이름 넣어 짧은 이야기 생성
  - 고학년 자유입력 R6 위반 시 대안 제시 ("무기 → 빛의 방패")

> [!note] 와우 포인트
> "반짝이는 숲에서 수아가 꽃을 모으고 있었어요. 작은 새가 말했어요. '수아, 넌 이 숲에서 가장 멋진 마법사야.'" 자기 이름이 주인공인 이야기를 처음 듣는 순간.

**AI 호출 2**: 이름 삽입 이야기 생성 (세계 + 활동 + 아이 이름 → 맞춤 단편 이야기)

**완성 보장 (R4)**: 18분 경과 미완성 → 현재 내용 + 자동 엔딩

---

### 휴식 (14:25–14:35, 10분)

순수 휴식. 아이들 캐릭터 둥둥 떠다니는 Idle Screen + BGM. 퍼실리테이터: 미완성 아이 catch-up 타임.

---

### 블록 4 — 꾸미기 + 완성 (14:35–14:55, 20분)

- **형식**: 개인
- **활동**:
  - 표지 분위기 선택 → AI 표지 이미지 생성
  - 제목 짓기
  - 최종 결과물: 표지 + 제목 + "지은이: OOO" + 이야기

> [!note] 와우 포인트 — 핵심
> 병원 차트엔 "환자명: 박수아, 진단: 백혈병". 근데 이 그림책엔 "지은이: 박수아". 몇 년 뒤 다시 펼치면 아이가 병원에서 웃었던 날을 기억한다.

**AI 호출 3**: 표지 이미지 생성 (분위기 + 세계 선택 → 표지 일러스트)

**완성 보장 (R4)**: 종료 5분 전 전원 체크 → 미완성 = 템플릿 즉시 투입 ("요정들이 OO이 대신 멋진 표지를 완성해 줬어!")

---

### 블록 5 — 발표회 (14:55–15:15, 20분)

- **형식**: 전체 · 대형 스크린
- **활동**:
  - 조별 대표작 1개씩 (총 8개) 대형 스크린 표시
  - 자원자 한 줄 소개 + 전체 박수

> [!note] 와우 포인트
> "25번 환자"가 아니라 "박수아, 반짝이는 숲의 마법사"로 불린다. 모든 부모가 박수 치고, 아이가 웃고, 부모가 운다.

---

### 블록 6 — 수료식 (15:20–15:30, 10분)

- **형식**: 전체
- **활동**:
  - QR → 갤러리 URL (40명 전원 작품)
  - 수료증: "마법사 OOO" + 작품 제목
  - 단체사진

> [!note] 와우 포인트
> 모든 검사 결과와 약물 기록은 병원에 남지만, 이 그림책은 집에 간다. "내가 만들었잖아, 엄마. 나 마법사였잖아."

## 안전 체계 4종

### 완성 보장 (R4)
3단계 체크: 블록2 15분 / 블록3 18분 / 블록4 종료 5분 전. 소외감 없는 포장 멘트 필수. [[product-requirements]] R4 항목과 직결.

### 콘텐츠 필터 (R6)
시스템프롬프트 + 2차 필터 + 선택지 구조적 차단. 위반 시 거절이 아닌 대안 제시 (예: "무기 → 빛의 방패"). [[product-requirements]] R6 항목.

### 에러 처리 (R5)
"마법사가 쉬고 있어요" + 재시도 버튼. 기술 용어 절대 노출 금지. [[no-debug-philosophy]] 원칙과 일치. [[product-requirements]] R5 항목.

### 퍼실리테이터 원칙
기술 문제 = 즉시 개입. 창작 결정 = 질문으로만 유도, 대신 선택 절대 금지. 아이의 자기결정권 보호.

## v0.3 vs pilot-curriculum-adapted 비교

> [!note] 구조 차이
> [[pilot-curriculum-adapted]] (2026-04-12 초안)은 게임 제작 중심의 6블록 구성. v0.3은 **그림책/창작 서사 중심**으로 전환되었으며, 게임 대신 표지+이야기 결과물을 생산한다. 핵심 변화:
> - 결과물 형식: 게임 HTML → **그림책 (표지 + 이야기 + 지은이)**
> - 출력 공유: 개별 발표 → **갤러리 URL + 수료증**
> - 콘텐츠 서사: 게임 플레이 → **"내가 주인공인 이야기"**
