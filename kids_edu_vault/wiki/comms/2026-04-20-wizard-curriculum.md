# 2026-04-20 — 나만의 마법 게임 만들기: 지휘관의 탄생

**스토리 아크**: 병원(지시받는 곳) → 오늘은 내가 지시하는 자 → 내 캐릭터/세계 창조 → 증거(QR + 사원증) 손에 쥐고 복귀

---

## 핵심 재정의: "게임 빌드"

> 아이가 묘사한 캐릭터 + 세계 + 규칙을 AI가 받아 **게임 타이틀 카드 페이지**로 즉시 생성.
> 실제 조작 게임이 아님. 캐릭터 이미지·세계관 텍스트·승리 조건·개발자 이름이 한 화면에 완성되는 것.

이전 v0.3([[2026-04-19-curriculum-v0.3]])는 그림책 결과물 중심이었으나, 본 설계는 **게임 타이틀 카드** 중심으로 재정의. 상세 스펙: [[curriculum-wizard-v1]]

---

## 사전 파악된 문제점 5가지

| 등급 | 문제 | 해결 방향 |
|---|---|---|
| 🔴 | "즉시 플레이 가능한 게임 빌드" 기술 미존재 | 게임 타이틀 카드로 재정의 |
| 🔴 | 40명 동시 진행 — 병동 집합 허가 문제 | 병원 사전 승인 필수 |
| 🔴 | 동아일보 취재 — 보호자 서면 동의 필요 | 병원 홍보팀 통해 사전 협의 |
| 🟡 | 연령 범위 미정 (0~18세) | 대상 연령 확정 후 조정 |
| 🟡 | 기기 40대 + Wi-Fi 준비 주체 미확정 | 명시 필요 |

## 타임테이블 (13:30–15:30)

| 블록 | 시간 | 내용 |
|---|---|---|
| 세팅 | 13:00–13:30 | 기기·Wi-Fi·스크린 준비 (운영팀) |
| 1 | 13:30–13:45 | 지휘관의 선언 (나쁜 명령 vs 좋은 명령 시연) |
| 2 | 13:45–14:05 | 아바타 소환 (캐릭터 묘사 → 이미지 카드 생성) |
| 3 | 14:05–14:25 | 세계 구축 (배경·승리 조건 → 게임 타이틀 카드 완성) |
| — | 14:25–14:35 | 갤러리 타임 (슬라이드쇼 자동 순환) |
| 4 | 14:35–15:00 | 마스터 개발자 임명식 (QR 확인 + 사원증 수령) |
| 5 | 15:00–15:20 | 런칭쇼 (전체 슬라이드쇼 + 미디어 기록) |
| — | 15:20–15:30 | 마무리·퇴장 |

## 기술팀 사전 구현 목록

| 항목 | 우선순위 |
|---|---|
| 웹 인터페이스 (캐릭터 묘사 → AI 이미지 → 게임 카드 자동 빌드) | 🔴 필수 |
| 갤러리 기능 (완성 카드 서버 저장 → 스크린 슬라이드쇼) | 🔴 필수 |
| QR 생성 (개인 게임 카드 URL → QR 자동 생성) | 🔴 필수 |
| 사원증 출력 (이름 + QR → 인쇄 가능 PDF) | 🟡 필요 |
| 오류 대비 캐시 (AI 오류 시 사전 생성 결과 이미지 세트) | 🟡 필요 |
| 부하 테스트 (40명 동시 AI 요청 처리 확인) | 🔴 필수 |

## 당일 운영 리스크 대응

| 상황 | 대응 |
|---|---|
| 특정 아이 진도 느림 | 진행자 1명 전담, 힌트 버튼 유도 |
| AI 응답 지연/오류 | 30초 내 미해결 시 사전 캐시 결과 표시 |
| Wi-Fi 불안정 | 핫스팟 백업 2대 즉시 전환 |
| 아이 활동 거부 | 강요 없이 관람자 참여, 갤러리 타임 합류 유도 |
| 미디어 촬영 혼선 | 동의 아이 좌석에 스티커 표시 (운영팀만 인지) |
| 부모 대신 조작 시도 | 진행자: "지휘관이 직접 해야 해요" |
