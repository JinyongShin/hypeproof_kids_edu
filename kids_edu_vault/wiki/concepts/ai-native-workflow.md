# AI 네이티브 워크플로우

## 정의
- 작업의 기본 단위를 **사람→AI 프롬프트→검증** 흐름으로 설계하고, 모든 입출력을 텍스트·멀티모달로 기록하여 자동화 가능하게 만드는 작업 방식.

## 왜 중요한가
- 기록된 워크플로우 = 조직의 자산. 다음 번엔 해당 단계를 에이전트가 자동화 가능.
- AI 생성물을 인간이 **안목**으로 검증 — AI는 사용자를 만족시키려 해서 틀린 답도 확신있게 제시 ([[2026-01-19-meeting]]).

## 구성 요소 / 하위 개념
- **추상화 → 구체화 → 딥 리서치 프롬프트**: 기본 프롬프트 구조 ([[2026-01-05-meeting]]).
- **인사이트 템플릿**: 롤 모델 지정("당신은 X처럼 생각해야 한다") → 결과 품질 향상.
- **요약 vs 인사이트 분리**: AI 요약부와 인간 생각부를 명시적으로 구분.
