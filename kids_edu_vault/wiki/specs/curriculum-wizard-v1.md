## 목표 경험

```
병원 = 지시받는 곳
        ↓
오늘은 내가 지시하는 자
        ↓
내 캐릭터 창조 → 내 세계 창조 → 세상에 공개
        ↓
증거(QR + 사원증)를 손에 쥐고 복귀
```

## 핵심 재정의: "게임 빌드"란?

아이가 묘사한 캐릭터 + 세계 + 규칙을 AI가 받아 **게임 타이틀 카드 페이지**로 즉시 생성.
실제 조작 게임이 아님. 캐릭터 이미지 · 세계관 텍스트 · 승리 조건 · 개발자 이름이 한 화면에 완성.
이것이 "플레이 가능한 게임"처럼 느껴지도록 UI 설계 필요.

---

## 전제 문제 5가지

| 등급 | 문제 |
|---|---|
| 🔴 | "즉시 플레이 가능한 게임 빌드" — 기술 미존재 → 타이틀 카드로 재정의 |
| 🔴 | 40명 동시 집합 — 병동 허가 문제 (병원 사전 승인 필수) |
| 🔴 | 동아일보 취재 — 미성년자 의료 환경 촬영 동의 (보호자 서면 동의 필수) |
| 🟡 | 연령 범위 미정 (0~18세) — 대상 연령 확정 후 세부 조정 필요 |
| 🟡 | 기기 40대 + Wi-Fi — 준비 주체 미확정 |

## 블록 1: 지휘관의 선언 (13:30–13:45, 15분)

**포맷**: 진행자 주도, 대형 스크린, 전체 관람

### 오프닝 스크립트
> "오늘 이 시간엔 선생님도, 부모님도 명령 못 해요.
> 여기서 명령하는 사람은 여러분뿐이에요.
> 그런데 AI는 멍청해서 — 여러분이 정확하게 말해줘야만 움직여요."

### 시연 순서
| 순서 | 입력 | 기대 반응 |
|---|---|---|
| ① 나쁜 명령 | "멋진 기사 만들어줘" | "에이~" |
| ② 좋은 명령 | "은빛 갑옷을 입고 왼쪽 눈에 흉터가 있는 여자 기사, 밤하늘 별빛 아래 서 있어" | "와!" |
| ③ 선언 | "더 구체적으로 말할수록 AI가 굴복해요. 여러분이 지휘관입니다." | — |

> ⚠️ 리스크: 인터넷 지연 시 → 사전 캡처 이미지 슬라이드로 즉시 대체

---

## 블록 2: 아바타 소환 (13:45–14:05, 20분)

**포맷**: 개인 기기, 개별 진행

### 웹앱 화면 흐름
```
[화면 1] "당신의 캐릭터는 어떻게 생겼나요?" → 자유 입력
[화면 2] AI 유도 질문 (미리 설계, 실시간 AI 아님) → 입력
[화면 3] 캐릭터 이미지 + 묘사 텍스트 카드 생성 → [다음 단계로]
```

### 진행자 원칙
- ❌ 절대 금지: 대신 입력해주기, 정답 알려주기
- ✅ 허용: 질문으로 유도, 선택지 제시
- 막힌 아이: "지휘관님, 이 캐릭터 머리카락은 어떤 색이에요?"
- 타이핑 느린 아이: 옆에서 불러주면 아이가 직접 입력

---

## 블록 3: 세계 구축 (14:05–14:25, 20분)

**포맷**: 개인 기기, 개별 진행

### 웹앱 화면 흐름
```
[화면 1] "당신의 캐릭터가 사는 세계는 어디인가요?"
  힌트 버튼: [우주] [바닷속] [마법 숲] [미래 도시] [직접 입력]
[화면 2] "이 세계에서 이기려면 무엇을 해야 하나요?"
[화면 3] "게임이 만들어지고 있습니다..." (로딩 3초)
[화면 4] 게임 타이틀 카드 완성 → [완성] 버튼 → 서버 저장 + 갤러리 자동 반영
```

### 게임 타이틀 카드 구성
```
┌──────────────────────────────┐
│  [게임 제목]                  │
│  [세계 배경 이미지]            │
│  캐릭터: [이름 / 묘사]         │
│  미션: [승리 조건]             │
│  총괄 개발자: OOO             │
└──────────────────────────────┘
```

---

## 갤러리 타임 (14:25–14:35, 10분)

- 완성된 게임 카드 5초 간격 자동 순환, 이름 크게 표시
- 진행자: "저기 OOO 지휘관 것 나온다!" 포인팅
- 미완성 아이: 이 시간 동안 마무리 가능

---

## 블록 4: 마스터 개발자 임명식 (14:35–15:00, 25분)

| 시간 | 내용 |
|---|---|
| 14:35–14:45 | 게임 카드 최종 확인 + 1회 재생성 허용 |
| 14:45–14:55 | 개인별 QR코드 화면 표시 → 아이가 직접 스캔 |
| 14:55–15:00 | "마스터 개발자 OOO" 사원증 + 실물 카드 수령 |

### 사원증 구성
```
┌─────────────────────────────┐
│  MASTER DEVELOPER           │
│  [아이 사진 또는 캐릭터 이미지] │
│  이름: OOO                  │
│  작품: [게임 제목]            │
│  날짜: 2026.5.5             │
│  [QR 코드]                  │
└─────────────────────────────┘
```

## 블록 5: 런칭쇼 (15:00–15:20, 20분)

- 전체 슬라이드쇼 재상영 (2회전 ≈ 7분)
- 진행자가 원하는 아이 지목해 한마디 (강요 아님)
- 동아일보 취재: 동의서 제출한 아이만 촬영
- 타이밍: 40명 × 5초 × 2회전 = ~7분 + 인터뷰 5명 이내 = 20분 내 마무리 가능

## 기술 구현 요구사항

| 항목 | 우선순위 |
|---|---|
| 웹 인터페이스 (캐릭터 묘사 → AI 이미지 생성 → 게임 카드 자동 빌드) | 🔴 필수 |
| 갤러리 기능 (완성 카드 서버 저장 → 스크린 슬라이드쇼 자동 반영) | 🔴 필수 |
| QR 생성 (개인 게임 카드 URL → QR 자동 생성) | 🔴 필수 |
| 사원증 출력 (이름 + QR 자동 배치 → 인쇄 가능 PDF) | 🟡 필요 |
| 오류 대비 캐시 (AI 오류 시 사전 생성 결과 이미지 세트) | 🟡 필요 |
| 부하 테스트 (40명 동시 AI 요청 처리 확인) | 🔴 필수 |
