# 상품 요구사항 (Product Requirements)

JY/Ryan 실행환경 공통 요구사항. 파일럿(2026-05-05) 기준.  
원본: `meeting_notes/production_requirements.md`

---

## R1. 진입 허들 제로

아이가 자리에 앉으면 **30초 안에 채팅 시작** 가능해야 한다.  
회원가입·로그인·앱 설치가 아이 앞에 노출되면 안 된다.  
운영자가 사전 세팅 완료, 아이는 화면을 열면 바로 시작.

> 현황: `?child=01` URL로 진입 — 로그인 없음. ✅

## R2. 타이핑 최소화

소아암 환아는 피로도가 높고 타이핑이 느리거나 힘든 경우가 많다.  
**선택지 버튼, 프롬프트 카드** 등 타이핑 없이도 진행 가능한 경로 필수.  
타이핑은 "하고 싶은 아이가 하는 것"이지 "해야만 진행되는 것"이 아니다.

> 현황: PromptScaffold 카드 존재. 카드 클릭 → 입력창 채움 → 전송 버튼 클릭 2단계. 🔶 원클릭 전송 검토 필요.

## R3. 첫 결과 빠르게

첫 입력(또는 첫 버튼 클릭) 후 화면에 **뭔가 움직이는 게 빨리** 보여야 한다.  
"로딩 중" 상태가 길어지면 아이는 흥미를 잃는다.

> 현황: WebSocket 스트리밍 — 텍스트 즉시 시작. ✅

## R4. 완성 보장

2시간 종료 시 **40명 전원이 "내 작품"** 을 가지고 있어야 한다.  
중간에 막히거나 에러 → 빈 화면으로 끝나는 아이가 있으면 실패.  
최악의 경우에도 기본 템플릿 위에 아이 이름이 들어간 결과물이 나오는 **안전장치** 필요.

> 현황: Claude 실패·타임아웃 시 빈 iframe. ❌ 폴백 템플릿 미구현.

## R5. 아이에게 코드/에러 노출 금지

에러 메시지·코드·기술 용어가 아이 화면에 나오면 안 된다.  
뭔가 잘못되면 **친근한 메시지**로 대체.

> 현황: HTML 코드 블록 화면 제거, 에러는 한국어 친근 메시지. ✅

## R6. 콘텐츠 안전

AI 생성 콘텐츠에 **폭력·전투·체력 소거·죽음 관련 요소 금지**.  
소아암 환아 앞에서 "생명/죽음/HP 감소" 게임 메카닉은 부적절.  
협력형·탐험형·창작형으로 제한 (꽃 피우기, 별 모으기, 길 만들기 등).

> 현황: TUTOR.md 페르소나에 명시적 제한. ✅ → [[combat-vs-cooperative-framing]]

## R7. 대형 스크린에 작품 표시

클로징 때 **조별 대표작 8-10개**를 선별해 대형 스크린에 소개.  
40개 전부가 아닌 퍼실리테이터가 조별 1개씩 선택해 발표.  
나머지는 갤러리 URL로 보호자에게 전달 (R8 연결).

> 현황: 갤러리 뷰 없음. ❌ 퍼실리테이터용 어드민 페이지 필요.

## R8. 결과물 가져가기

워크샵 종료 후 아이(또는 보호자)가 **작품을 가져갈 수 있어야** 한다.  
QR코드, 짧은 URL, 또는 스크린샷 이미지 중 하나.

> 현황: 공유 기능 없음. ❌ 게임 HTML 저장·공유 메커니즘 미구현.

## R9. 촬영 친화적 화면

화면이 **시각적으로 밝고**, 카메라에 담겼을 때 "아이가 AI로 뭔가 만들고 있다"가 읽혀야 한다.  
텍스트만 가득한 어두운 화면은 기사 사진으로 쓸 수 없다.

> 현황: `bg-gray-950` 다크 테마. ❌ 밝은 테마·큰 UI 개편 필요.
