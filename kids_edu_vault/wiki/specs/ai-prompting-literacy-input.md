# AI 프롬프팅 리터러시 — 커리큘럼 인풋

> BH의 4/19 커리큘럼 초안에 반영 요청하는 인풋 spec.
> "아이가 게임을 만드는 경험"과 "AI에게 잘 시키는 법을 체득"을 동시에 달성하기 위한 레이어.

---

## 왜 필요한가

Scratch 같은 기존 도구는 "코딩 문법"을 가르친다.
이 파일럿의 차별점은 **"AI와 협력하는 법"** — 결과물보다 **대화 방식**이 핵심 스킬이다.

소아암 병동 맥락에서:
- 치료 이후 사회에서 쓸 수 있는 미래 스킬
- 타이핑·코딩 능력 없어도 익힐 수 있음
- "내가 말해서 만든 것"이라는 주도성·자기효능감

---

## 블록별 프롬프팅 스킬 매핑

기존 [[pilot-curriculum-adapted]] 6블록 구조를 유지하면서, 각 블록에 프롬프팅 스킬 1개를 추가.

| 블록 | 기존 활동 | 추가할 프롬프팅 스킬 | 좋은 예 | 나쁜 예 |
|---|---|---|---|---|
| **Block 0** (10분) | 캐릭터 고르기 | **묘사하기** — 원하는 걸 말로 그리기 | "날개 달린 토끼 캐릭터 만들어줘" | "캐릭터 만들어" |
| **Block 1** (15분) | 움직이기 | **구체화** — 막연 → 구체 | "왼쪽 화살표 누르면 왼쪽으로 움직여, 속도는 빠르게" | "움직여" |
| **Block 2** (15분) | 친구 찾기 | **추가 요청** — 기존에 덧붙이기 | "거기다 친구 캐릭터도 화면 오른쪽에 추가해줘" | "친구도 넣어" |
| **Block 3** (15분) | 아이템 모으기 | **수정 요청** — 특정 부분 바꾸기 | "별 대신 하트로 바꿔줘, 색은 분홍색으로" | "바꿔줘" |
| **Block 4** (15분) | 나만의 마법 | **자유 조합** — 배운 4패턴 자유 활용 | (제한 없음, 자유 입력) | — |
| **Block 5** (15분) | 보여주기 | **언어화** — 어떻게 말했는지 설명 | "나는 이렇게 말했더니 이렇게 됐어요" | — |

## 운영 방식 제안

### 퍼실리테이터 역할 추가
각 블록 시작 전 퍼실리테이터가 **"오늘의 말하기 미션"** 1개 공지.

예: "Block 1 미션: 오늘은 '어떤 키를 누를 때'와 '얼마나 빠르게'를 꼭 넣어서 말해봐요!"

### 블록 전환 시 1분 회고
블록 끝나기 1분 전, 퍼실리테이터가 아이에게:
"AI한테 뭐라고 말했어요? 어떻게 말했더니 잘 됐어요?"

### 성취 인정
Block 4에서 자유롭게 프롬프팅 → Block 5에서 "내가 어떻게 말했는지" 설명하면
→ **"AI 크리에이터 인증"** 수료 포인트로 삼을 수 있음.

## 제품 UI에 필요한 기능 (JY → JeHyeong 전달)

### 1. 프롬프트 스캐폴딩 카드
- 현재 블록(Block 0~4)에 맞는 예시 문장 카드 3~4개
- 카드 클릭 → 입력창 자동 채우기
- Block 4는 카드 없음 (자유 입력 유도)

### 2. "잘 됐던 말들" 히스토리
- AI가 게임을 성공적으로 만들어준 경우, 그 입력 문장을 자동 저장
- 세션 내 히스토리 패널에 표시
- "아까 이렇게 말했더니 됐었어" 참고용

### 3. Claude 메타 피드백
Claude 시스템 프롬프트에 지시:
응답 마지막에 한 줄 추가 → "💡 다음엔 [더 구체적인 표현 예시]라고 해봐!"

## Claude 시스템 프롬프트 요구사항

[[ai-persona-workflows]]의 W3(빠른 구현) + W4(스토리텔링) 합성 페르소나 기반.

추가 규칙:
1. 게임을 **즉시** 만들어줄 것 — 설명보다 완성이 먼저
2. 코드는 절대 보여주지 말 것 — HTML만 반환
3. 전투·체력 소거, 협력형 서사만 (꽃 피우기, 별 모으기, 길 만들기, 힐링)
4. 응답 마지막에 **반드시** "💡 다음엔 ~라고 해봐!" 1줄
