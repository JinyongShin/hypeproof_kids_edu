# 커리큘럼 스킨 — 성인 (Adult)

> 코어 흐름: [[curriculum-core]] 참조.
> 이 문서는 성인 대상 스킨 — 컨텍스트·스크립트·결과물만 정의.
> 첫 케이스: 치과 스태프 (보아치과, 2026-05-26 예정)

---

## 컨셉

> "원장님은 5분, 직원이 주인공"

원장님의 신뢰도는 오프닝 5분에만 쓴다.
나머지는 직원이 자기 업무에서 직접 프롬프트를 만들고 저장해서 나간다.

---

## 스킨 교체 요소

| 요소 | 성인 스킨 값 |
|---|---|
| 오프닝 프레임 | 동료/상사가 실제 쓰는 사례를 먼저 보여준다 |
| 탐색 대상 | 자기 업무 중 반복·귀찮은 것 1개 |
| 첫 시도 결과물 | 내 업무에 맞는 프롬프트 첫 버전 |
| 개선 기준 | "맥락을 넣으면 AI가 더 잘한다" |
| 최종 결과물 | 자기 도구에 저장된 프롬프트 1개 |
| 페르소나 파일 | `TUTOR_adult.md` |
| 언어 수준 | 직무 용어 사용 가능, 설명 최소화 |

---

## 사전 확인 (워크샵 전)

| 항목 | 확인 방법 |
|---|---|
| 오프닝 시연자가 실제로 쓰는 AI 사례 1개 | 사전 컨택: "어떤 업무에 쓰세요?" 확인. 블록 1에서 그걸 그대로 씀 |
| 참여자 직군 및 주요 반복 업무 | 블록 2 막힐 때 대비 선택지 준비 |
| AI 계정 | 1인 1계정 (공유 계정 불가) |

---

## 블록별 스크립트

### 블록 1 — 도입: 시연 (5분)

**오프닝 시연자**: 원장님 또는 AI를 이미 쓰는 동료

> "저는 이걸 이렇게 씁니다." (설명 없이 실행)

- 직원들이 화면을 본다
- 진행자가 직원들에게만 질문: **"어떠세요? 해보고 싶으세요?"**

> ⚠️ 시연자가 이 이후에도 계속 주도하려 하면:
> "이번엔 직원분들 차례예요. 지켜봐 주세요."

---

### 블록 2 — 탐색: 내 업무 고르기 (10분)

> "매일 또는 매주 하는 일 중에 가장 귀찮거나 시간이 걸리는 것 하나만 고르세요."

시연자(원장님 등)는 이 블록에 참여하지 않는다.

**막혔을 때 직군별 선택지 (진행자 카드)**

| 직군 | 예시 |
|---|---|
| 위생사 | 진료 후 주의사항 설명 / 스케일링 안내 문자 |
| 코디네이터 | 예약 안내 문자 / 첫 방문 환자 안내 / 리뷰 답글 |
| 사모님 | 직원 인수인계 메모 / 공지 작성 |
| (다른 직군) | 진행자가 사전에 해당 직군용 예시 준비 |

---

### 블록 3 — 첫 시도: 그냥 해보기 (10분)

> "지금 생각나는 대로 AI한테 말해보세요. 잘 하려고 하지 마세요."

시연자는 이 블록에서 절대 개입하지 않는다.
진행자가 사전 고지: "이 구간은 직원들이 스스로 해봐야 하니, 말씀 아껴주세요."

결과가 나오면:
- 만족 → "그게 오늘 첫 번째 버전이에요. 잘 기억해두세요."
- 불만족 → "어떤 부분이 달랐어요?" → 블록 4로

---

### 블록 4 — 개선: 해부 + 힌트 (15분)

직원의 첫 결과를 같이 뜯는다. 시연자는 **힌트 제공자**로만 등장.

**순서**
1. 진행자 → 직원: **"AI가 뭘 몰랐을까요?"**
2. 직원이 스스로 답하게 한다 (맥락 부족, 말투 안 맞음 등)
3. 답이 안 나오면 → 시연자에게: "이 결과 보시기에 뭐가 빠진 것 같으세요?"
4. 시연자는 힌트만 ("환자 나이대를 넣어보면 어떨까요?" — 직접 쓰지 않음)
5. 직원이 그 힌트를 받아 직접 다시 입력
6. 두 결과 나란히 비교 → 개선 폭 가시화

**시간 부족 시**: 해부 질문 1개만 하고 블록 5로

---

### 블록 5 — 공유: 저장 + 서로 보여주기 (10분)

**저장**: 각자 자기 프롬프트를 자기 도구에 저장
- 카카오톡 나에게 보내기 / 메모앱 / 노션 등
- 저장 형식: 프롬프트 텍스트 + "언제 쓰는지" 한 줄 메모

**공유**: 서로 보여주기
- 시연자가 "오, 이거 바로 써도 되겠는데요" → 최고의 마무리

**클로징 스크립트**
> "오늘 각자 하나씩 만들어 가셨어요.
> 핵심은 세 가지입니다.
> 1. 맥락을 넣으면 AI가 더 잘한다
> 2. (시연자)도 처음엔 이렇게 시작했다
> 3. 이걸 반복하면 업무마다 하나씩 쌓인다
>
> 오늘 만든 거, 이번 주에 한 번만 실제로 써보세요."

## 퍼실리테이터 — 시연자 관리

| 상황 | 대응 |
|---|---|
| 블록 3에서 직원 대신 타이핑 시도 | "직원분이 직접 해봐야 해요. 잠깐만요." |
| 블록 4에서 정답 말해버림 | "맞아요, 근데 OO씨가 직접 넣어볼게요." |
| 블록 5에서 직원 결과 평가 시작 | 긍정 반응만 유도: "이거 바로 쓸 수 있겠죠?" |

**직원 동기 저하 시**
> "처음엔 다들 이래요. (시연자)도 처음엔 이랬을 거예요."
→ 시연자에게 확인 요청: "처음에 어떠셨어요?"

## 준비물

| 항목 | 필수 | 비고 |
|---|---|---|
| 시연자 실제 사례 1개 | ✅ | 블록 1용. 사전 확인 필수 |
| AI 도구 (ChatGPT / Claude) | ✅ | 참여자 1인 1계정 |
| 노트북 또는 태블릿 | ✅ | 직원 1인 1대 |
| 직군별 반복 업무 예시 목록 | ✅ | 블록 2 막힐 때 진행자 카드 |
| 프로젝터 / 대형 모니터 | 권장 | 결과 비교 시 전체 공유용 |
