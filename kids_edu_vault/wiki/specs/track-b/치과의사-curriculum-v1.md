# 치과의사 AI 실전 워크샵 v1

## 사전 확인 사항 (워크샵 전)

| 항목 | 확인 방법 | 미충족 시 대응 |
|---|---|---|
| 원장님 AI 사용 경험 여부 | 사전 컨택 시 확인 | 미경험자면 블록 3을 진행자가 직접 시연으로 대체 — 원장님 대신 "저희가 써봤더니..." 방식으로 전환 |
| 참여 인원 및 직군 구성 | 사전 확인 | 원장님만 있고 임직원 없으면 블록 3+스테이션 2 역할 구도 재조정 필요 |

---

## 블록 1: 일상 묻기 (10분)

한 분씩 돌아가며 "오늘 하루 어떤 일 하셨어요?"
- 원장님: 진료, 카톡봇 관리, 강의 준비 등
- 위생사: 스케일링, 환자 관리, 예약 확인 등
- 코디네이터: 예약 관리, 환자 응대, 서류 등
- 사모님: 자유

이 단계에서 AI 얘기 전혀 안 함. 그냥 **어떻게 일하는지**만 듣는다.

## 블록 2: "왜 그렇게 하세요?" (10분)

블록 1에서 나온 일 중 하나를 골라서 왜 그렇게 하는지 묻는다.
- "그건 왜 손으로 하세요?"
- "그건 왜 매번 반복하세요?"
- "그건 왜 종이에 적으세요?"

→ 본인도 모르게 **불편함**이 튀어나옴

여기서 나온 불편함이 **이하 모든 스테이션의 소재**가 됨. 하나의 불편함을 끝까지 밀어붙인다.

## 블록 3: "AI로 하면 어떻게 될까요?" (10분)

원장님께: "이거 AI로 하면 어떻게 될까요?"
- 원장님이 이미 AI를 쓰는 사람이라서 설명 가능
- 임직원 앞에서 원장님이 설명 → **"우리 원장님이 이런 거 하셨어?"** 와우
- 임직원은 "나도 저런 거 쓸 수 있나?" 궁금해짐

## 블록 4: 스테이션 — 하나의 불편함을 끝까지 (20~40분)

블록 2에서 뽑은 **불편함 하나**를 가지고 스테이션을 순서대로 통과한다.
스테이션 1·2는 **필수**, 스테이션 3·4는 시간 여유에 따라 선택 진행.

| 스테이션 | 소요 | 필수 여부 |
|---|---|---|
| 1 전심전력 | 10분 | ✅ 필수 |
| 2 역목표 설계 | 10분 | ✅ 필수 |
| 3 만족 유의 | 5~10분 | 선택 (시간 있을 때) |
| 4 잇기 가설 | 5~10분 | 선택 (시간 있을 때) |

### 스테이션 1 — 전심전력 (10분) ✅ 필수
- "그 불편함을 AI에게 어떻게 말하면 가장 잘 알아들을까요?"
- 원장님+임직원이 같이 프롬프트를 작성
- 처음엔 대충 쓰다가 → "더 자세히 말해볼까요?" → 입력의 해상도가 올라감
- **포인트**: 대충 쓴 결과 vs 정성껏 쓴 결과를 비교해서 보여줌

### 스테이션 2 — 역목표 설계 (10분) ✅ 필수
- 스테이션 1에서 나온 결과를 가지고 — "이거 실제로 도입하면 뭐가 제일 위험할까요?"
- **임직원**이 대답: "환자한테 이상한 말 하면 큰일 나요", "의료정보라 조심해야 해요"
- 원장님보다 **현장 리스크를 임직원이 더 잘 앎**
- 위험을 다시 AI에게 물어봐서 방어책까지 만듦

### 스테이션 3 — 만족 유의 (5~10분) _(선택)_
- 스테이션 2에서 찾은 리스크를 반영해서 — "이거 충분히 좋은가요? 한 번 더 밀어볼까요?"
- 같은 프롬프트를 다른 각도로, 다른 모델로, 또는 조건을 추가해서 재시도
- **개선 폭**이 눈에 보임 → "아, 한 번 더 하니까 더 좋아지네"

### 스테이션 4 — 잇기 가설 (5~10분) _(선택)_
- 개선된 결과를 가지고 — "이걸 다른 업무에도 연결할 수 있을까요?"
- 원장님이 스스로 확장: "예약 관리에도 쓸 수 있겠다", "환자 안내에도..."
- **스스로 가능성을 발견**하는 순간

## 블록 5: 클로징 (10분)

"원장님, 방금 4개 원칙으로 문제를 푸셨습니다"
1. 전심전력 — 입력의 해상도를 끌어올리기
2. 역목표 설계 — 위험을 먼저 찾기
3. 만족 유의 — 한 번 더 밀어붙이기
4. 잇기 가설 — 다른 곳에도 연결하기

"이런 원칙들이 더 있습니다. 이걸로 체계화하시면 치과의사들한테 강의하실 수 있어요."

## 퍼실리테이터 가이드

### 전체 원칙
- ❌ 절대 금지: 정답 먼저 주기, 진행자가 대신 입력하기
- ✅ 허용: 질문으로만 유도, 침묵 3초 이상 지속되면 선택지 제시
- 임직원이 먼저 말하도록 유도 — 원장님 단독으로 흐르지 않게

### 스테이션 1 동선 (전심전력)
1. 먼저 대충 쓰게 놔둔다. 개입하지 않는다.
2. 결과가 나오면: *"어떠세요? 생각하신 것과 비슷한가요?"*
3. 만족스럽지 않다는 반응이 나오면: *"어떤 부분이 다른가요?"*
4. 스스로 더 구체화하지 못하면: *"예를 들어, 어떤 상황에서 쓰시나요? 어떤 환자한테?"*
5. 개선된 프롬프트 입력 → 결과 나오면 두 결과를 나란히 비교

### 스테이션 2 동선 (역목표 설계)
1. 결과를 보여주며 **원장님이 아닌 임직원에게 먼저** 질문: *"이게 실제로 쓰이면 현장에서 가장 걱정되는 게 뭐예요?"*
2. 임직원 답변을 충분히 듣는다 — 원장님이 끊지 않도록
3. 그 위험을 가지고: *"그럼 AI한테 이 걱정도 같이 말해줄 수 있을까요?"*

### 스테이션 3 동선 (선택)
- *"지금 이 결과, 충분한 것 같으세요? 한 번만 더 눌러볼까요?"*
- 재시도 후 개선 폭이 느껴지면 바로 다음으로

### 스테이션 4 동선 (선택)
- 원장님에게: *"이 방식, 다른 업무에도 쓸 수 있을 것 같으세요?"*
- 직접 말이 나오도록 기다린다. 10초 침묵해도 괜찮다.

## 준비물

| 항목 | 필수 여부 | 비고 |
|---|---|---|
| AI 도구 | ✅ 필수 | ChatGPT Plus 또는 Claude Pro 권장. 참여자 계정 사전 생성 또는 진행자 계정 공유 방식 중 선택 |
| 노트북 또는 태블릿 | ✅ 필수 | 참여자 1인 1대. 치과 기기 사용 가능하면 별도 준비 불필요 |
| 프로젝터 또는 대형 모니터 | 권장 | 결과 비교 시 전체가 함께 보기 위함. 없으면 진행자 화면 공유로 대체 |
| 참여 인원 사전 파악 | ✅ 필수 | 원장님 외 직군 구성 확인 (위생사/코디/사모님 여부) |
