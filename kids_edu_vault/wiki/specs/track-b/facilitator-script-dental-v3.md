# 진행자 스크립트 — 치과의사 워크샵 v3

> 이 스크립트는 외울 필요 없음. 흐름을 알고, 굵은 글씨 대사만 자연스럽게 꺼내면 됨.
> `[지시]` = 진행자 행동. `"..."` = 실제 발화.

---

## 입장 전 (D-10분)

```
[지시] 원장님과 단둘이 30초 대화
```

> "원장님, 오늘 AI 직접 쓰시는 사례 하나 보여주실 거 있으세요? 5분이면 충분해요."

원장님이 사례 말하면:
> "딱 그거 하나만 해주세요. 설명은 안 하셔도 됩니다. 그냥 실제로 쓰는 것 보여주시면 돼요."

```
[지시] 직원들 자리 배치 확인 — 각자 기기 앞에 앉아 있는지
[지시] AI 도구 로그인 미리 완료 확인
```

---

## 블록 1: 원장님이 먼저 보여주기 (5분)

```
[지시] 전체에게
```

> "오늘 딱 한 시간이에요. 끝나면 각자 하나씩 만들어서 나가실 거예요. 뭘 만드는지는 잠깐 뒤에 알게 되실 거고요."

> "시작 전에, 원장님이 실제로 쓰시는 거 잠깐 보여주실 거예요. 구경만 하시면 돼요."

```
[지시] 원장님에게 신호 → 원장님 시연
[지시] 진행자는 말 없이 직원들 표정 관찰
```

시연 끝나면 원장님이 아닌 **직원들에게만** 질문:

> **"어떠세요? 해보고 싶으세요?"**

```
[지시] 반응 기다림. 최소 5초.
```

반응이 나오면 그대로 받고 다음 블록으로.
반응이 없으면:
> "해보면 알아요. 지금 바로 해보실 거니까."

```
[지시] 원장님에게
```
> "원장님, 이제부터는 직원분들 차례예요. 지켜봐 주세요."

---

## 블록 2: 내 업무 고르기 (10분)

```
[지시] 전체에게
```

> **"매일 또는 매주 반복하는 일 중에, 가장 귀찮거나 시간 걸리는 거 하나만 골라주세요. 딱 하나만요."**

> "정답 없어요. 크든 작든 상관없어요. 그냥 본인한테 귀찮은 거면 돼요."

```
[지시] 1분 침묵. 개입하지 않음.
```

고르지 못하는 사람이 있으면 개별 접근:
> "OO씨는 하루에 몇 번 반복하는 게 있어요? 제일 많이 하는 거요."

그래도 못 고르면 직군별 선택지 제시:
> "혹시 환자 안내 문자라든가, 진료 후 설명이라든가... 이런 거 중에 있어요?"

```
[지시] 각자 고른 업무 확인 (간단히 한 바퀴 돌며)
```
> "OO씨는요? / OO씨는요?"

---

## 블록 3: 첫 번째 시도 (10분)

```
[지시] 원장님에게 먼저 (조용히, 1:1로)
```
> "원장님, 이 구간은 직원분들이 스스로 해봐야 하는 구간이에요. 말씀 아껴주시면 훨씬 효과가 크거든요."

```
[지시] 전체에게
```

> **"지금 생각나는 대로 AI한테 말해보세요. 잘 하려고 하지 마세요. 틀려도 돼요."**

> "지금 이 결과가 별로여도 괜찮아요. 그게 다음 단계 재료가 되거든요."

```
[지시] 각자 타이핑. 진행자는 뒤에서 화면만 슬쩍 확인. 개입 없음.
```

결과 나오기 시작하면 한 명씩:
> "어떠세요?"

**만족하는 경우**:
> "그럼 그게 오늘 첫 번째 버전이에요. 잘 기억해두세요. 이거 곧 더 좋아질 거예요."

**불만족하는 경우** (이게 더 좋음):
> "어떤 부분이 달랐어요?"

대답 들은 다음:
> "맞아요. 그게 블록 4에서 고칠 부분이에요. 그대로 두세요, 잠깐 있다가 같이 봐요."

---

## 블록 4: 해부 + 원장님 힌트 (15분)

```
[지시] 직원 중 결과가 가장 아쉬운 사람부터 시작. 또는 자원자.
```

> **"이 결과, 같이 뜯어봐요."**

### 해부 질문 1 — 맥락

> **"AI가 뭘 몰랐을까요? 이 결과가 어색한 이유가 뭘까요?"**

```
[지시] 10초 기다림.
```

직원이 답하면 그대로 받음:
> "맞아요. 그 정보를 AI가 몰랐던 거예요."

답이 안 나오면 **원장님에게**:
> "원장님, 이 결과 보시기에 어떤 정보가 빠진 것 같으세요?"

원장님 힌트 나오면 직원에게 바로 전달:
> "OO씨, 원장님 말씀 들으셨죠? 그 부분 추가해서 다시 한번 해볼게요?"

```
[지시] 직원이 직접 수정해서 재입력
[지시] 두 결과 나란히 보이게
```

> **"차이 보이세요?"**

### 해부 질문 2 — 역할 (시간 있을 때)

> **"AI한테 어떤 역할을 주면 더 잘할까요? 예를 들어 '너는 친절한 치과 코디야' 같은 거요."**

직원이 직접 생각해서 추가 → 재입력 → 비교.

---

## 블록 5: 저장 + 공유 (10분)

```
[지시] 전체에게
```

> **"이제 저장할게요. 오늘 이거 안 저장하면 없어져요."**

> "카카오톡 나에게 보내기, 메모앱, 어디든 편한 곳에요. 저장하실 때 한 줄만 추가해주세요 — '이거 언제 쓰는 건지'."

```
[지시] 다 저장하면 공유 시간
```

> **"각자 뭐 만드셨는지 한 마디씩만요."**

```
[지시] 한 바퀴 돌며 짧게. 원장님도 마지막에.
```

원장님 반응 유도:
> "원장님, 직원분들 거 보시니까 어떠세요?"

---

## 클로징

> **"오늘 각자 하나씩 만들어서 저장하셨어요."**

> "핵심은 세 가지예요."
> "첫 번째, 맥락을 넣으면 AI가 더 잘해요."
> "두 번째, 원장님도 처음엔 이렇게 시작하셨을 거예요."
> "세 번째, 이거 반복하면 업무마다 하나씩 쌓여요."

> **"이번 주에 딱 한 번만 실제로 써보세요. 그것만 해주시면 돼요."**

```
[지시] 끝. 박수 유도하지 말 것. 조용히 마무리.
```

## 긴급 상황 대응

| 상황 | 대응 |
|---|---|
| 직원이 타이핑 자체를 거부 | "같이 해볼게요. 제가 불러드릴 테니까 OO씨가 타이핑만 해주세요." |
| 원장님이 계속 개입 | "원장님, 지금 직원들 반응 관찰하시는 게 더 중요해요. 나중에 피드백 주세요." |
| 첫 결과가 너무 좋아서 할 게 없음 | "완벽하네요. 그럼 더 어려운 업무 하나 더 해볼게요?" |
| 시간이 너무 지체됨 | 블록 4 해부 질문 1개만 → 바로 블록 5 저장으로 |
| AI 오류 / 느림 | "이런 경우도 있어요. 다시 한번 해볼게요." → 그냥 재시도 |
