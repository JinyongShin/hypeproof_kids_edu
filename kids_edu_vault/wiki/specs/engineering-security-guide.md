   - 유지보수 불필요 (정적 페이지)

5. **40대 부하테스트**
   - API rate limit 대응: 큐잉 + 순차 처리

### 🟡 필요 (4/30 리허설 전)

## 데이터 보안 정책

### 병원에 사전 요청할 것
- 아이 이름 (사원증 + 타이틀 카드용)
- 연령대

### PII 규칙

| 항목 | LLM 프롬프트 | 저장 | 비고 |
|------|-------------|------|------|
| 이름 | ✅ OK | ✅ OK | PII 아님 |
| 연령 | ✅ OK | ✅ OK | |
| 진단명 | ❌ 절대 금지 | ❌ | 의료정보 |
| 병동 정보 | ❌ 절대 금지 | ❌ | 의료정보 |
| 치료 내용 | ❌ 절대 금지 | ❌ | 의료정보 |
| 주민번호 | ❌ | ❌ | |
| 연락처 | ❌ | ❌ | |

### 프롬프트 가이드라인
- AI에게 "소아암 병동의 아이가 ~" 같은 컨텍스트 절대 금지
- 아이가 입력하는 내용(캐릭터, 세계, 미션)만 프롬프트에 포함
- 저장 데이터: 이름 + 생성결과만. 진단명/병동 정보 제외

### 당일 아이 사진
- 촬영 동의서 제출한 아이만 촬영 (TJ 피드백 확인 완료)
- 사원증에 아이 사진 vs 캐릭터 이미지: 병원 확인 필요

## "게임" 정의 (헷갈림 방지)

커리큘럼에서 "게임" = **게임 타이틀 카드** (이미지+텍스트 레이아웃)

```
┌──────────────────────────────┐
│  [게임 제목]                  │
│  [세계 배경 이미지]            │
│  캐릭터: [이름 / 묘사]         │
│  미션: [승리 조건]             │
│  총괄 개발자: OOO             │
└──────────────────────────────┘
```

실제 조작 가능한 게임은 **스콥 밖**. QR이 가리키는 것도 타이틀 카드 정적 페이지.
