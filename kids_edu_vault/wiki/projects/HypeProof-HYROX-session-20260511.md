## Phase 1: 스테이션 설계 (봉호 태님과)

### 16원칙 HYROX 스테이션 구성
4랩 × 4스테이션, 1세션 = 1시간 (도입 10분 + 실습 40분 + 회고 10분)

| 주차 | 테마 | 스테이션 |
|---|---|---|
| 1주 | Foundation | 전심전력 / 잇기가설 / 질문공터 / 입력굴리기 |
| 2주 | Load | 부하걸기 / 만족유의 / 역할몰입 / 역목표 |
| 3주 | Mastery | 백번뽑기 / 다중모델 / 추상사다리 / 수행위임역전 |
| 4주 | Transcendence | 언러닝 / 상상 / 소격 / 천번째감탄 |

### 3라인업 체제
| 라인업 | 타겟 | 시간 | 핵심 가치 |
|---|---|---|---|
| A. Explorer | 초·중학생 | 2시간 | 창조의 기쁨 |
| B. Builder | 전문직·성인 | 1시간 | 업무 해결 |
| C. Racer | HYROX 랭킹 참가자 | 1시간×4주 | 체력 증명, Verify 뱃지 |

공통 엔진: 16원칙 + 도입→실습→클로징 + CoT 감사. 예시만 교체 (One engine + Multi skin).

### 커리큘럼 vs 프로덕션 경계
- **커리큘럼** = PDF로 출력 가능 (내용, 예시, 루브릭, 진행자 가이드)
- **프로덕션** = 브라우저에서 작동 (플랫폼, 랭킹, 결제, 자동채점)
- 현재: 커리큘럼 100%, 프로덕션 0%

### 채점 루브릭 (100pt 만점)
| 차원 | 배점 | 평가 질문 |
|---|---|---|
| 입력 해상도 | 25 | AI에게 얼마나 구체적으로 말했는가? |
| 반복 심화 | 25 | 첫 결과에서 멈췄는가, 더 밀었는가? |
| 비판적 감사 | 25 | 모델 출력을 비판적으로 검토했는가? |
| 연결·확장 | 25 | 이 원칙을 다른 맥락에 연결했는가? |

### 이중 평가 구조
- **내재 평가 (자동)**: 입력 횟수, 길이 변화, 수정 이력, 소요 시간, 모델 전환
- **외부 평가 (진행자)**: 맥락 이해도, 감정 전이, 품질 판단, 협업 질
- 비율: 2.5 자동 / 1.5 사람 = 진행자 수고 60% 절감

### 참여자 Takeaway
- 즉시: 본인 산출물, 원칙 카드, 프롬프트 템플릿팩
- 1주 내: 세션 리포트
- 4주 완주: Verify 뱃지, 종합 역량 프로필, 커뮤니티 입장권
- B2B: 팀 단위 리포트, 인증 패키지

### MVP 상태
PDF 3장 = MVP 전부. 4~5시간 작업.

## Phase 2: 프로덕트 피칭 & 본질 정합 (주인님과)

### 주인님 피드백 요약
1. "코드블락 쓰지 말고 핵심만" → 불필요한 기술적 묘사 제거
2. "진행자를 왜 없애?" → GTM이 KOL 기반인데 진행자 없애면 KOL이 설 자리 사라짐
3. "볼트를 보고 대답해라" → 비즈니스 전략, 채널 구조를 먼저 확인하고 주장해야
4. "겉만 보지 말고 본질이 같은 걸 이어봐" → 두 채널(비트리/다이렉트)이 따로 노는 게 아니라 HYROX의 체육관/대회 관계

### 확정된 프로덕트 정의
**16원칙 HYROX = "AI를 쓰는 사람들의 경기장"**
- 참가자가 스테이션에서 AI와 실습 → 자동 채점 → 점수 + 랭킹 + 뱃지
- 진행자 필수: KOL이 진행자로 서는 게 바이럴의 핵심
- 자동 채점 = 진행자를 돕는 도구 (진행자를 대체하지 않음)
- 확장: 봉호 태님 → KOL(보아치과 원장님) → Racer 완주자가 진행자 자격 취득

### 두 채널의 본질적 연결
- **비트리 채널** (바이오팜, 동아일보) = HYROX의 **체육관 제휴** (훈련)
- **다이렉트 채널** (보아치과) = HYROX의 **대회** (경쟁·랭킹)
- 체육관 → 대회 → 다시 체육관. 이 루프가 경쟁 기반 커뮤니티.

### 수익 모델 (HYROX 대응)

|---|---|---|
| 참가비 | $96M (69%) | 스테이션 참가비 (4주코스) |
| 라이선스 | 체육관당 $1,500/년 | 진행자(KOL) 연회비 |
| 굿즈/사진 | 피니시라인 사진 | Verify 뱃지, 역량 리포트 |
| B2B | 관람 티켓 | 팀 단위 리포트 |

## Phase 3: HYROX 리버스 엔지니어링

### Moritz Fürste의 사고 과정
1. **빈 곳을 발견**: 헬스 다니는 사람은 많은데 경기장이 없다 → "AI 쓰는 사람은 많은데 경기장이 없다"
2. **당연함을 의심**: "헬스는 취미지 스포츠가 아니다"를 뒤집음 → "AI 교육은 강의다"를 의심
3. **측정→재방문**: 마라톤 기록이 재방문을 만듦 → 16원칙 점수가 재방문을 만듦
4. **파트너십**: Moritz(운동선수) + Toetzke(이벤트 베테랑) → 봉호 태님(16원칙) + 주인님(비즈니스 설계)
5. **프로덕트 안에 바이럴**: 피니시라인 사진 = 참가자가 스스로 홍보 → Verify 뱃지 SNS 공유

### 핵심 통찰
- HYROX의 성장은 광고가 아니라 프로덕트 안에 설계된 전파 메커니즘
- 표준화 = 비교 = 랭킹 = 재방문 = 커뮤니티 = 자가 확장
- 우리의 우위: 물리적 인프라(트럭 9대) 없이 웹앱으로 한계비용 ≒ 0
