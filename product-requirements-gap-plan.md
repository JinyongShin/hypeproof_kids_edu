## P0 — 리허설 전 필수 (4/26 이전)

### [P0-A] R4: 폴백 템플릿 (완성 보장)

**문제**  
Claude 실패·타임아웃 시 게임 영역이 빈 화면으로 남는다. 아이가 아무 결과물 없이 워크샵을 끝낼 수 있다.

**구현 방향**

1. **백엔드: 기본 템플릿 폴백**  
   `claude_runner.py`의 에러·타임아웃 분기에서 `StreamEvent(type="game", html=FALLBACK_HTML)` 추가 발송.  
   `FALLBACK_HTML` = 아이 이름을 파라미터로 받아 표시하는 간단한 별 모으기 게임.  
   `TUTOR.md`의 Mock 응답과 동일한 구조로 충분.

2. **프론트엔드: "한 번 더 해볼까?" 메시지**  
   에러 이벤트 수신 시 게임 영역에 기본 템플릿 자동 삽입 + 채팅창에 `"잠깐! AI가 졸았나봐. 다시 한번 말해봐! 🔁"` 표시.

3. **게임 HTML에 아이 ID 삽입**  
   `stream_claude`에서 `child_id`를 게임 HTML 제목에 주입하는 간단한 문자열 치환.  
   예: 폴백 게임 제목 `<title>child01의 별 모으기</title>`.

**변경 파일**
- `src/backend/claude_runner.py` — FALLBACK_HTML 상수 + 에러/타임아웃 분기에 game 이벤트 추가
- `src/backend/tests/test_claude_runner.py` — 폴백 발송 테스트 추가

### [P0-B] R9: 밝은 UI 테마

**문제**  
현재 전체 배경 `bg-gray-950` 다크 테마. 카메라 촬영 시 "어두운 채팅 화면"으로 보임.  
기사·홍보 사진에서 "아이가 AI로 게임 만드는 장면"이 전달되지 않는다.

**구현 방향**

1. **배경**: 채팅 영역 `bg-white` / `bg-slate-50`, 게임 영역은 iframe이 게임 색상을 그대로 표시하므로 변경 불필요.

2. **텍스트**: 어두운 색(`text-gray-800`)으로 전환. 사용자 메시지 버블은 인디고 유지.

3. **글씨 크기**: 프롬프트 카드·메시지를 `text-base` 이상으로 키움. 아이 눈높이 + 카메라 가독성.

4. **PromptScaffold 카드**: 현재 작은 `text-xs` 버튼 → `text-sm` 이상 + 더 큰 패딩, 밝은 배경.

5. **블록 진행 표시**: 숫자만 있는 버튼 → 스킬 이름 포함 ("묘사하기", "구체화" 등) 표시. 퍼실리테이터가 보기에도 현재 단계가 바로 보여야 함.

6. **전송 버튼**: `▶` 텍스트 → 크고 명확한 버튼 ("보내기" 또는 아이콘).

**변경 파일**
- `src/frontend/app/globals.css` — 기본 배경색
- `src/frontend/app/layout.tsx` — body 배경
- `src/frontend/components/ChatPane.tsx` — 색상·폰트 크기 전면 수정
- `src/frontend/components/PromptScaffold.tsx` — 카드 크기·색상

## P1 — 파일럿 전 필요 (5/5 이전)

### [P1] R8: 결과물 가져가기

**문제**  
워크샵 종료 후 아이/보호자가 게임을 가져갈 방법이 없다.

**구현 방향 (단계적)**

**옵션 A (단순, 리허설 때 검증)**: 게임 HTML을 백엔드에 파일로 저장 + `/game/{child_id}` 엔드포인트로 서빙.  
보호자에게 `hypeproof-ai.xyz/game/child01` URL 전달. QR코드는 운영자가 별도 생성.

**옵션 B (자동화)**: 게임 완성 시점에 자동으로 서버 저장 → 프론트엔드에 "QR 보기" 버튼 표시.  
QR 라이브러리(`qrcode` 또는 Google Charts API) 로 즉시 생성.

> 옵션 A로 리허설 진행, 옵션 B는 파일럿 전에 결정.

**변경 파일**
- `src/backend/main.py` — `GET /game/{child_id}` 엔드포인트 추가
- `src/backend/claude_runner.py` — game 이벤트 발송 시 HTML을 `data/games/{child_id}.html`로 저장
- `src/frontend/components/GamePreview.tsx` — "링크 복사" 또는 "QR 보기" 버튼

## P2 — 있으면 좋음 (퍼실리테이터 편의)

### [P2] R7: 대형 스크린 갤러리

**문제**  
클로징 때 조별 대표작을 한 화면에서 보여줄 방법이 없다.  
현재는 퍼실리테이터가 아이 화면을 직접 전환해야 함.

**구현 방향**

`/admin/gallery` 페이지: 저장된 `data/games/*.html`을 불러와 썸네일 그리드로 표시.  
각 썸네일 클릭 시 풀스크린 iframe. 비밀번호 보호 또는 URL 비공개로 운영.

> R8(게임 HTML 저장)이 선행되어야 구현 가능.

**변경 파일**
- `src/frontend/app/admin/gallery/page.tsx` — 갤러리 페이지 (신규)
- `src/backend/main.py` — `GET /admin/games` 엔드포인트 (저장된 game 목록)

## R2 개선 (선택)

**카드 원클릭 전송**: 현재 카드 클릭 → 입력창 채움 → 전송 버튼. 2단계.  
`PromptScaffold`의 `onSelect`를 `onSend`로 바꾸면 카드 클릭 즉시 전송 가능.

**트레이드오프**: 실수 클릭 시 취소 불가. 아이가 카드 내용을 확인 후 전송하는 현재 방식이 교육적으로 더 나을 수 있음.  
→ 리허설에서 퍼실리테이터 피드백 후 결정.
